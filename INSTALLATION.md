# Rent CMS - Installation & Setup Guide\n\n## Overview\n\nRent CMS is a comprehensive property management system built with PHP and MySQL. It includes role-based access control for Admin, Tenant, and Guest users, with features for property listing, searching, and inquiry management.\n\n## System Requirements\n\n- **PHP**: 7.4 or higher\n- **MySQL**: 5.7 or higher\n- **Web Server**: Apache or Nginx\n- **File Permissions**: Write access to `/uploads` folder\n\n## Installation Steps\n\n### 1. Database Setup\n\n1. Create a MySQL database:\n   ```sql\n   CREATE DATABASE rent_cms;\n   ```\n\n2. Import the database schema:\n   ```bash\n   mysql -u root -p rent_cms < database.sql\n   ```\n\n   Or through phpMyAdmin:\n   - Create database `rent_cms`\n   - Import `database.sql` file\n\n### 2. Configuration\n\n1. Update database credentials in `config/database.php`:\n   ```php\n   define('DB_HOST', 'localhost');      // Your database host\n   define('DB_USER', 'root');           // Your database username\n   define('DB_PASS', '');               // Your database password\n   define('DB_NAME', 'rent_cms');       // Database name\n   ```\n\n### 3. File Structure\n\n```\nrent-cms/\n‚îú‚îÄ‚îÄ admin/                    # Admin panel files\n‚îÇ   ‚îú‚îÄ‚îÄ dashboard.php        # Admin dashboard\n‚îÇ   ‚îú‚îÄ‚îÄ users.php            # User management\n‚îÇ   ‚îú‚îÄ‚îÄ properties.php       # Property management\n‚îÇ   ‚îú‚îÄ‚îÄ inquiries.php        # Inquiry management\n‚îÇ   ‚îî‚îÄ‚îÄ settings.php         # System settings\n‚îú‚îÄ‚îÄ assets/\n‚îÇ   ‚îú‚îÄ‚îÄ css/\n‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ style.css        # Main stylesheet\n‚îÇ   ‚îú‚îÄ‚îÄ js/                  # JavaScript files\n‚îÇ   ‚îî‚îÄ‚îÄ images/              # Static images\n‚îú‚îÄ‚îÄ config/\n‚îÇ   ‚îî‚îÄ‚îÄ database.php         # Database configuration\n‚îú‚îÄ‚îÄ includes/\n‚îÇ   ‚îú‚îÄ‚îÄ auth.php             # Authentication functions\n‚îÇ   ‚îú‚îÄ‚îÄ header.php           # Navigation header\n‚îÇ   ‚îî‚îÄ‚îÄ footer.php           # Footer component\n‚îú‚îÄ‚îÄ pages/                   # Public pages\n‚îÇ   ‚îú‚îÄ‚îÄ index.php            # Homepage\n‚îÇ   ‚îú‚îÄ‚îÄ login.php            # Login page\n‚îÇ   ‚îú‚îÄ‚îÄ register.php         # Registration page\n‚îÇ   ‚îú‚îÄ‚îÄ about.php            # About page\n‚îÇ   ‚îú‚îÄ‚îÄ contact.php          # Contact page\n‚îÇ   ‚îú‚îÄ‚îÄ profile.php          # User profile\n‚îÇ   ‚îú‚îÄ‚îÄ property-detail.php  # Property detail\n‚îÇ   ‚îú‚îÄ‚îÄ add-property.php     # Add property\n‚îÇ   ‚îú‚îÄ‚îÄ my-properties.php    # Manage properties\n‚îÇ   ‚îú‚îÄ‚îÄ edit-property.php    # Edit property\n‚îÇ   ‚îî‚îÄ‚îÄ delete-property.php  # Delete property\n‚îú‚îÄ‚îÄ uploads/                 # User uploaded images\n‚îú‚îÄ‚îÄ database.sql             # Database schema\n‚îî‚îÄ‚îÄ README.md               # This file\n```\n\n### 4. Create Required Folders\n\n```bash\nmkdir -p uploads\nchmod 755 uploads\n```\n\n### 5. Web Server Configuration\n\n#### For Apache:\n\nCreate `.htaccess` in root:\n```apache\n<IfModule mod_rewrite.c>\n    RewriteEngine On\n    RewriteCond %{REQUEST_FILENAME} !-f\n    RewriteCond %{REQUEST_FILENAME} !-d\nRewriteRule ^(.*)$ index.php?q=$1 [QSA,L]\n</IfModule>\n```\n\n#### For Nginx:\n\nAdd to your server block:\n```nginx\nlocation / {\n    try_files $uri $uri/ /index.php?$args;\n}\n```\n\n## User Roles & Features\n\n### Guest (Public User)\n- Browse all approved properties\n- Search and filter by city, category, and deal type\n- View property details\n- Submit inquiries about properties\n- No authentication required\n\n### Tenant (Registered User)\n- All Guest features\n- Register and login to system\n- Create and publish property listings\n- Upload multiple property images\n- Manage own properties (edit, delete)\n- View inquiries received\n- Update profile information\n\n### Admin (Administrator)\n- All Tenant features\n- Manage all users (activate/deactivate)\n- Approve/reject property listings\n- Delete any property or user\n- View all inquiries\n- Manage system settings\n- View analytics and statistics\n\n## Default Admin Account\n\nTo create an admin user, insert directly into database:\n\n```sql\nINSERT INTO users (username, email, password, first_name, last_name, role, status) \nVALUES ('admin', 'admin@rentcms.com', '$2y$10$YourHashedPasswordHere', 'Admin', 'User', 'admin', 'active');\n```\n\nOr use PHP to generate password hash:\n```php\n<?php\necho password_hash('admin123', PASSWORD_BCRYPT);\n?>\n```\n\n## Features Overview\n\n### Homepage\n- Hero section with background image\n- Search filters (City, Deal Type, Category)\n- Grid display of popular listings\n- Property cards with images and pricing\n\n### Property Listing\n- Title, description, address, location\n- Price with flexible units (per month, per day, per sq ft)\n- Bedrooms, bathrooms, square footage\n- Amenities list\n- Multiple image gallery\n- View counter\n- Owner contact information\n\n### Inquiry System\n- Contact property owners\n- View inquiry status (new, read, replied, archived)\n- Email notifications (can be configured)\n\n### Admin Dashboard\n- Statistics cards (users, properties, pending items)\n- Quick access to pending properties\n- User management\n- Property approval workflow\n- Inquiry management\n- System settings\n\n## Security Features\n\n- Password hashing with bcrypt\n- Session-based authentication\n- SQL injection prevention with prepared statements\n- XSS protection with sanitization\n- Role-based access control\n- CSRF token support (can be added)\n\n## File Upload\n\n- Maximum file size: Configure in `php.ini`\n- Allowed formats: JPG, PNG, GIF\n- Files stored in `/uploads` folder\n- Unique filenames to prevent conflicts\n\n## Troubleshooting\n\n### Database Connection Error\n- Verify database credentials in `config/database.php`\n- Check MySQL server is running\n- Ensure database user has proper permissions\n\n### Page Not Found\n- Verify .htaccess or Nginx configuration\n- Check file paths are correct\n- Ensure all files are uploaded\n\n### Upload Issues\n- Check `/uploads` folder permissions (755)\n- Verify `upload_max_filesize` in `php.ini`\n- Check available disk space\n\n### Login Issues\n- Clear browser cookies/cache\n- Verify user email and password\n- Check database has user records\n\n## Database Structure\n\n### Tables\n\n1. **users** - User accounts and profiles\n2. **properties** - Property listings\n3. **property_images** -
